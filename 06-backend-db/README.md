# 06 Backend & DB

## Mission

This stage turns file-and-function fluency into application-shaped backend work.

By the end of this stage, a learner should be able to:

- connect application code to SQL databases intentionally
- write safer query and transaction flows
- recognize repository boundaries and where they help
- read and extend small HTTP and client-side examples

## Stage Map

| Track | Surface | Core Job |
| --- | --- | --- |
| `BD.1` | [web and database](./01-web-and-database) | teach database/sql, repository boundaries, HTTP examples, and broader backend reference surfaces |

## Why This Stage Exists Now

The learner already knows how to model data, move text, and work across files.
Now the curriculum turns that knowledge into code that looks and feels like a real application.

## Suggested Learning Flow

1. Start with the [databases track](./01-web-and-database/databases).
2. Use the repository milestone as the main proof surface.
3. Explore the HTTP client, migrations, and web-masterclass material as supporting depth.
